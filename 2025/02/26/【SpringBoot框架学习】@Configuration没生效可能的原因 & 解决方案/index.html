    问题：
    Spring Boot 只会自动扫描启动类所在包及其子包，如果你的 @Configuration 类在其他包中，就不会被加载。

    问题：
    @Configuration 需要被 Spring 容器管理，否则不会生效。

    解决方案：

    • 确保 没有 用 new 手动创建 @Configuration 类对象。
